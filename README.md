# debian-server-simulation
Debian 12 üzerinde Apache (Web), Bind9 (DNS), VSFTPD (FTP) ve Subdomain yapılandırmalarını içeren tam kapsamlı sunucu simülasyonu.
# Debian Üzerinde Web (Apache), DNS (Bind9), Subdomain ve FTP Sunucusu Kurulumu Proje HakkındaBu proje, Bursa Teknik Üniversitesi "Sunucu İşletim Sistemi" dersi kapsamında hazırlanmıştır. Projenin amacı, Debian tabanlı bir sunucu işletim sistemi üzerinde statik IP yapılandırması, Web, DNS, Subdomain ve FTP servislerini barındıran gerçek bir hosting altyapısı simülasyonu oluşturmaktır.🚀 Kurulan Servisler ve Mimari Yapı* İşletim Sistemi ve Ağ: Sanal makine üzerine Debian 12 işletim sistemi kurulmuştur. Güvenli ve izole bir yapı sağlamak amacıyla ağ modu "Host-Only Adapter" olarak seçilmiş ve IP adresi 192.168.56.101 olarak statik ayarlanmıştır.
* Web Sunucusu (Apache): Ana alan adı olan miracbayoglul.com için Apache2 servisi kurulmuştur. Gelen istekler özel VirtualHost yapılandırmasıyla doğrudan /var/www/miracbayoglul.com dizinine yönlendirilmektedir.
* DNS Sunucusu (Bind9): İstemcilerin isim çözümlemesi yapabilmesi için Bind9 DNS servisi kurulmuştur. Zone (bölge) dosyasına SOA, NS ve A kayıtları girilerek alan adı sunucu IP adresine yönlendirilmiştir.
* Subdomain Yapılandırması: Ana siteden bağımsız olarak yazilim.miracbayoglul.com alt alan adı tanımlanmıştır. Bunun için Bind9 üzerinde A kaydı eklenmiş ve /var/www/yazilim dizini üzerinden farklı bir web sayfası yayına alınmıştır.
* FTP Sunucusu (VSFTPD): Dosya transferini kolaylaştırmak amacıyla kurulmuştur. Yapılandırma dosyasında write_enable=YES parametresi ile dosya yükleme yetkilendirmesi aktifleştirilmiştir.🛠️ Karşılaşılan Sorunlar ve Çözüm YollarıSistemin kararlılığını sağlamak için karşılaşılan teknik aksaklıklar şu şekilde çözülmüştür:Dosya Erişim İzni (403 Forbidden): FTP üzerinden atılan dosyalarda yaşanan erişim sorunu; chown ve chmod 755 komutları kullanılarak Apache kullanıcısına gerekli okuma izinleri verilerek çözülmüştür.
* Windows İstemcisi DNS Çözümleme Hatası: Windows istemcisinin kendi DNS'ini kullanması nedeniyle siteye erişememesi sorunu, Windows Ağ Bağdaştırıcısı IPv4 ayarlarından manuel DNS olarak 192.168.56.101 atanarak giderilmiştir.
* Subdomain Yönlendirme Çakışması: Subdomain'e gidildiğinde ana sayfanın açılması engellenerek, Apache VirtualHost konfigürasyonunda ServerName tanımlanmış ve DocumentRoot yolu özelleştirilmiştir.👨‍💻 Geliştirici* Muhammet Miraç Bayoğlu * Tarih: 02.01.2026
